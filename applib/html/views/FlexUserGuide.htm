Flex Introduction

Gamification can be a powerful tool for contact center leaders who want to heighten awareness of a specific metrics, boost individual or team engagement, motivate the workforce to achieve a goal, or simply add some fun into the daily work routine. A-GAME Flex enables contact center supervisors and managers to create games for teams and agents in minutes. Game type, duration, theme, scoring basis, participants, and other setup options provide an almost endless platform from which to create engaging experiences that enhance agent support and improve workforce bonds. This guide will show you how to use A-GAME Flex and achieve consistent success with gamification.

Game Settings

The Game Editor

The Flex Game Editor is an intuitive wizard-style interface that makes creating and modifying games easy. To access the Flex Game Editor, log in to Acuity, click A-GAME | Flex | Add/Edit Games.

Most of the fields required in screen one should be self-explanatory.

  • Name. Provide a name for your game that's appropriate for your audience and whatever objective you have in mind to achieve.
  • Game Dates. Specify the Game's date range. While any range is possible, game durations longer than a month or shorter that a couple of days would be unusual. Use the date picker to specify the start and end dates for your game.
  • Theme — Type. There are two aspects of Theme to determine for your game.
    • First, involves the scoring methodology that will be used to measure and calculate game progress. Current options include Leaderboard, Points, and Phrase. Leaderboard games are the simplest to setup and understand, ranking participants in order of their standard or scored value to a specific KPI or metric. Points-based games award a defined number of points according to performance levels that will be defined in the next screen of the Game Editor. Phrase-based games enable you to create a multi-word phrase wherein players earn letters according to their performance. The first player or team that completes the phrase first is the game winner. Select from the options in the drop-down selector to specify the game type.
  • Theme — Theme. The Game Editor provides an evolving library of game themes to choose from. From the selector drop-down, select one that fits your objective. If you don't see what you're looking for or have a theme you'd like added, email support@tocuhpointone.con and our theme designers will take care of you.
  • Game Type. Flex Games can be created for Teams or Agents. Click the selector drop-down to indicate the appropriate Game Type.
    • Note: The Team Battle option supports Inter-Team competitions (games involving multiple teams), and soon, Intra-Team competitions. Intra-Team battles involve specific teams that are divided into sub-teams by the supervisor. If the Team Battle option is used, the Inter-Team (and soon Intra-Team) Sub-Group should be selected.
  • Game Admin. With Flex, games can be created by supervisors for their own teams or for supervisors by their managers. The Game Admin options available are managed as a function of the suer logged into the system. Logged in as a supervisor will allow supervisors to create games for their teams only. Managers to whom multiple supervisors report will be able to create games for any of their supervisors. Corporate user with higher privileges can create contests for supervisors across departments, projects, locations, etc.
    • Note: If you are logged into Acuity as Supervisor, your name will be the only one visible in the Game Admin drop-down selector. Managers can designate any supervisor that reports to him or her a Game Admin. Therefore, all supervisors that report to a the logged in manager are displayed in the drop-down selector.
  • Scoring Metric. Scoring metric is the basis on which game scoring will be calculated. Balanced Score or any primary KPI or sub-KPI configured within an agent scorecard can be used as the basis for game-scoring. A KPI that is not a part of any scorecard, called Custom Metric, can also be used for game scoring. To define the Scoring Metric, select from the listed options in the menu drop-down.
    • Note: If the selected KPI includes sub-KPIs, the relevant sub-KPI must also be selected.
    • Note: Because there is no Standard Value for Balanced Score, when it is selected for a Leaderboard game, the Scoring Basis is always Scored.
  • Scoring Basis. Acuity assimilates KPI data based on either the Standard or Scored value. Standard value is the native value provided by the source system. For example, call duration is typically measured in seconds, so a standard value for the Average Handle Time for a call might be 500 seconds (the standard value is 500). The Scored value is the standard value translated into a scored value, typically using a scale of 1-10. A call of 500 seconds duration might be great, mapping to a score of 10, or terrible earning a score of 1. Flex Games scoring can use either the Scored or Standard KPI value.
  • Allow Discussion Board? Flex games include an option to enable a game-centric bulletin board style communication feature for game participants to encourage, mentor, brag, boast, or smack-talk their fellow players.
  • Ending Points. Ending Points is a Flex game attribute that defines the number of points that a player must be awarded to win the game. The setting applies only to games that are points-based (see Theme — Type, #3 above).
    • Tip: The value for points-based games designed for a specific time duration, i.e., 30 days, should be set high enough to ensure players don’t win prematurely (for example, on day 5 of a 30-day game), but not so high that visualization of progress (be it via a race, puzzle, card, board, or other game metaphor) is indecipherable.
  • Status. The Status field indicates the Game’s current mode within the Flex system. When a new game is being created, the status is by default In Development until the setup is complete and the Game has been Started (status would change to Live) or the game has concluded (In Review), finalized (Final) and so on.
  • Game Phrase. The Game Phrase field will display the text for Phrase themed games (see section 3 above)
  • Scoring Model. Scoring Model options are determined by the selected Game Theme. For example, Leaderboard Games don’t use Custom Scoring, but this option used for Phrase and Points based games. The Creating Games section will provide instruction on how to create each game type.
  • Calculate Score. Flex Game scores can be calculated on either a Daily or Cumulative basis.

    Leaderboard games are inherently Cumulative, so this option is not displayed for that Game Type. A Points based game, however, might award points based on performance over a particular time-period — Cumulative, or on each day of the time period — Daily.

Creating Games

Leaderboard Game

Leaderboard Games are the most common type of Flex game and simple and fast to set up. Leaderboard Games can be, however, effective in a broad range of use cases. Following, we’ll consider one particular use to illustrate how to set up a Leaderboard Game.

Use Case: Supervisor wants to enhance agent support strategy using gamification. Game plan is to strengthen awareness and focus on overall Balanced Score by incorporating a one-month contest across all agents. Top three performing agents receive lunch (delivered!) the first business day following contest end. Top two most improved agents (performance over previous month’s score) receive same prize.

    To invoke the Game Editor Form, log in to Acuity, click A-GAME | Flex | Add/Edit Games. Then click the Add New Game button.

    The game Leaderboard updates in real-time and is accessible to the game administrator, managers, and participating employees (agents, CSRs, etc.) by clicking Flex in the menu. The leaderboard displays can be displayed on personal systems or public monitors or wallboards.
